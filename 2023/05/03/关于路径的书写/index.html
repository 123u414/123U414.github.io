关于路径的书写

大坑回填之文件路径书写

绝对路径

是个人都会写,拥有极差的可移植性,但是不排除某些重要功能需要使用它,在环境变量里我们常常见到 毕竟环境变量这东西要考虑什么可移植性呢 比如下面这个名为DriveData的环境变量,使用的是绝对路径,但需要注意环境变量中也有一些家伙使用了相对路径

C:\Windows\System32\Drivers\DriverData

相对路径

好吧,这才是今天要讲的大头,里面的注意事项和坑挺多的.

首先,最基本的相对路径,是酱婶的,它以某个资源作为参照物,描述另外一个资源的路径.这里不使用"文件"而使用"资源"这个概念,是因为我们不仅通过相对路径描述文件的位置,也用它来描述文件夹,网络资源等的路径.使用相对路径时可能遇到以下几个情况:

两个文件在同一个目录下

此时若以A资源的路径为参照物,描述和他处于同一个目录下的B资源,相对路径就直接写作

B

严谨一点写作

./B

./的意思是从同一级目录开始.

两个文件在不同目录下

同样以A资源为参照,描述B资源的位置,如果B在A的子目录下,其实跟绝对路径没有什么两样,只不过路径的起点变成了A的路径,使用./来代替,举个比方

./file1/file2/B

意思就是在A的同级目录中的file1的子目录中的file2中的B文件 感谢人类发明了相对路径

那么聪明的你的疑问中一定有个小脑袋:

如果B资源在A资源的父级目录怎么办呢?如果B资源在A资源的父级目录怎么办呢? 如果B资源在A资源的父级目录怎么办呢?

嗯哼,这个问题不止你一个人想到了,这个时候使用../代表A资源的父级目录就可以了.

什么?你说爷级目录?哦哦,父级目录的的父级目录是吧,这还不简单,../../搞定啦,每一个../就相当于一次"返回上一级"的操作.

最后,使用/表示根目录,这个根目录在不同的地方有不同的意义,具体要看使用的环境,应该谨慎使用.

好了,今天的文件路径就学到这…个屁,还有亿些坑没有讲完.

/ \ // \\ 四兄弟

/

首先是/,读作斜杠

最简单的路径分隔符,没什么花里胡哨的.传承自Unix的路径分隔符,所谓路径分隔符正统嫡传,根正苗红,而现在网络应用大多使用Unix及其后裔,所以几乎所有的网络地址都使用/作为分隔符.

在C语言的一系列文件操作函数中,也是使用/作为路径分隔符

    FILE * fp;
    fp = fopen("E:/test/file/data/d.txt","r");

\

然后是\,读作反斜杠

/如是说

诶?你是个什么异端分子,天生反骨,敢与我分隔符正统对着干?看我不把你c%!@FaJv81^@!%

/因言语过激被踢出小课堂

这个玩意是怎么来的呢,因为/已经被用作DOS的命令提示符的参数标志了,为了不引起混乱,Macrostiff™在其Widows®系统中使用\作为路径分隔符

//

再然后是//,读作双斜杠
他用来分隔协议和主机名.比如说

https://123u414.github.io

就是用//分隔http协议和主机名.

\\

最后是\\,读作双反斜杠
这个东西呢,也是一个怪胎,上文说过,Macrostiff™在其Widows®系统中使用\作为分隔符,但是呢,C语言使用\构成转义字符,这个时候,如果在字符串中使用\就会被编译器当做转义字符,产生一些问题.为了兼容\这个怪胎,于是人们又发明了\\这个双胞怪胎 屎山就此形成辣

通过上面的介绍,想必你也搞清楚了分隔符应该怎么使用,简单来说就是见人说人话,见鬼说鬼话,在谁的地盘用谁的分隔符.

最后

系统路径

路径名 意义 在书写这篇md的电脑上对应的绝对路径
%SystemDrive%(%Windir%) 系统安装的磁盘分区 C:\
%SystemRoot% 系统目录 C:\Windows
%ProgramFiles% 应用程序默认安装目录 C:\Program Files
%AppData% 应用程序数据目录 C:\Users\16528\AppData\Roaming
%HomePath% 当前活动用户目录 C:\Users\16528
%Temp%(%Tmp%) 当前活动用户临时目录 C:\Users\16528\AppData\Local\Temp
%DriveLetter% 逻辑驱动器分区 Null
%HomeDrive% 当前用户系统所在分区 C:\

系统路径可以被看做是一种特殊的相对路径的简写方式,只不过与一般的相对路径相比,它指示的是系统中重要资源的路径.
